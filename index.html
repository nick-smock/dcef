The Problem

A fragmented enterprise
without a common language.

"The U.S. counter-drone enterprise generates enormous amounts of test data. Almost none of it is comparable."

Every service, agency, and organization conducts counter-UAS evaluations independently — recording different metrics, using different formats, and storing results in isolated silos of inconsistent quality. The result is a community that is data-rich and information-poor.

The Close Combat Lethality Task Force FY25 Annual Report documented that all three military services continue to test commercial C-UAS solutions "independently in a disjointed manner." JIATF 401 was established in August 2025 with a marketplace for C-UAS acquisition — but that marketplace only works if every competition produces comparable, defensible data.

Without a standardized evaluation methodology, the government is spending money on competitions whose results cannot be compared, aggregated, or used to build the models that the modern threat environment demands.

The DCEF Framework

Three tiers. One standard.
Anywhere in the country.

DCEF's core technical contribution is using onboard drone telemetry — PX4 ULog and MAVLink tlog files from Pixhawk-based aircraft — as calibrated ground truth, correlated against radar, optical, and acoustic sensor outputs. The tier you use depends on what you're evaluating.

Tier 1
Initial Characterization
≤3m
CEP  ·  Circular Error Probable

Hardware: Standard GNSS (u-blox M8/M9 class)

Log format: PX4 ULog / MAVLink tlog

Infrastructure: Minimal — onboard only

Mass vendor screening. Up to 100+ systems evaluated against common flight profiles. Establishes baseline detection, tracking, and classification performance. Gateway to Tier 2.

Tier 2
Comparative Analysis
≤30cm
CEP  ·  Circular Error Probable

Hardware: Entry-level RTK (dual-band GNSS)

Log format: PX4 ULog / MAVLink tlog

Infrastructure: RTK corrections via NTRIP or base

Finalist comparison. Performance differentiation at the margins — where two systems score within 5% of each other in Tier 1, Tier 2 resolves the difference with data, not vibes.

Tier 3
High-Fidelity Assessment
≤5cm
CEP  ·  Circular Error Probable

Hardware: Full RTK — u-blox F9P class

Log format: PX4 ULog / MAVLink tlog + RTCM

Infrastructure: Fixed surveyed base station on range

Kinetic defeat evaluation. At this accuracy level, engagement geometry error is effectively eliminated — directly reducing rounds required to achieve a defined Pk at range. Tighter CEP means less ammunition.

Decision gates are mandatory. A system cannot advance from Tier 1 to Tier 2 evaluation without meeting defined threshold KPPs. Tier 3 requires Tier 2 passage. This is not a preference — it is a technical review requirement encoded in the DCEF SEP, aligned with DoD Manual 5000.100 and the Integrated Decision Support Key framework.
