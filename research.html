STEP 01
Ingest Telemetry

PX4 ULog and MAVLink tlog files are parsed using pyulog. Position, heading, speed, and RF link quality fields are extracted at native log rate.

STEP 02
Ingest Sensor Output

C-UAS sensor track data is ingested as CSV, mapped to the DCEF schema fields, and validated for completeness. Unmappable records are flagged.

DCEF Schema · CSV
→
STEP 03
Time Synchronize

Both streams are resampled to a common 1 Hz analysis clock. GPS-disciplined timestamps align records within the ±50ms tolerance window.

pandas · NTP · GPS time
→

Slant range, horizontal position error, detection flags, and range-bin assignments are computed per correlated record pair.

Visualization 01

2D Position Error Scatter

Each point represents a single correlated record — the sensor-reported target position plotted relative to the onboard telemetry ground truth at origin. The tighter the cluster around center, the higher the sensor's positional accuracy.

Points are color-coded by slant range band. This reveals whether sensor error grows with distance — a common radar characteristic — or remains consistent across the evaluation envelope.

Mean horizontal error: 4.2m — well within Tier 2 detection threshold
95th percentile error: 11.8m — driven by far-range outliers at 550–600m
Consistent northward bias visible across all range bands — indicates systematic antenna boresight offset
Position Error — Scenario Alpha · Radar Synthetic

Visualization 02

Polar Error Plot

The polar plot bins position error magnitude by vehicle heading at the time of detection. Each spoke represents the mean error for tracks where the drone was flying in that direction, revealing directional sensor bias that a simple mean error metric cannot capture.

A perfectly accurate sensor would produce a uniform circle. Elongation along any axis indicates the sensor over-reports or under-reports position in that direction — often caused by multipath, terrain masking, or antenna pattern irregularities.

Maximum bias at 340° (NNW) — 7.1m mean error, likely terrain masking from treeline
Minimum bias at 160° (SSE) — 2.8m mean error, clear line-of-sight corridor
Overall directional variance of 4.3m suggests site-specific siting adjustment recommended
Directional Bias — Scenario Alpha · Radar Synthetic

Visualization 03

Range-Binned Probability of Detection

The PoD curve plots the fraction of correlated truth records that produced a confirmed sensor detection, binned by 50m range intervals. The shaded band represents the 90% confidence interval derived from binomial sampling at each range step.

This is the primary procurement-relevant output of a DCEF evaluation. A PoD curve produced under DCEF methodology can be directly compared against curves from other sites and vendors — because the ground truth definition is identical.

PoD ≥ 0.90 maintained through 450m — meets Tier 2 detection threshold for this scenario
Curve knee at 480m — rapid detection degradation begins beyond this range
Wide confidence band at 550m+ indicates insufficient sample count — additional runs required at far range
Probability of Detection vs. Range — Scenario Alpha Synthetic

Visualization 04

Track Overlay / KML

The track overlay plots the telemetry ground truth flight path against the sensor-reported track for the same time window. Divergence between the two lines is position error made spatially visible — showing not just how much the sensor is wrong, but where and in what direction.

In the full DCEF pipeline this output is exported as a KML file, enabling direct overlay in Google Earth for site-specific context — terrain, structures, and sensor siting all visible alongside the track data.

Orange track — onboard telemetry ground truth (Tier 2 RTK, ≤30cm CEP)
Blue track — radar-reported sensor track for the same time window
Sensor track consistently leads truth track on inbound approach — indicates range-rate overestimation in the radar solution
Truth vs. Sensor Track — Scenario Alpha · Approach Profile Synthetic · KML Export Available
